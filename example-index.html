MoonNet Icon Pack - Usage Example

This page demonstrates the recommended way to implement favicons and the header logo for the MoonNet DevOps site.

Head Tags for Favicons

Add these tags to your <head> section:

<!-- SVG favicon (modern browsers) -->
<link rel="icon" type="image/svg+xml" href="icons/favicon.svg">

<!-- PNG fallback for older browsers -->
<link rel="icon" type="image/png" sizes="32x32" href="icons/generated/favicon-32x32.png">
<link rel="icon" type="image/png" sizes="16x16" href="icons/generated/favicon-16x16.png">

<!-- Apple Touch Icon for iOS home screen -->
<link rel="apple-touch-icon" sizes="180x180" href="icons/generated/apple-touch-icon.png">

<!-- Web App Manifest for PWA support -->
<link rel="manifest" href="site.webmanifest">

<!-- Theme color for mobile browser UI -->
<meta name="theme-color" content="#0A1F44">

Header Logo Implementation

The header logo should be displayed at 80px height by default:

<header>
    <a href="index.html" class="logo-link" aria-label="Moonnet DevOps home">
        <img 
            src="icons/logo.svg" 
            alt="Moonnet DevOps" 
            class="logo-img"
            height="80"
        >
    </a>
</header>

Generating Raster Icons

Run the build script to generate PNG, ICO, and WebP variants:

cd icons
./make_icons.sh

The script requires ImageMagick and optionally uses pngquant for optimization and cwebp for WebP conversion.

Notes

  • The Apple Touch Icon uses a navy (#082a4b) background for iOS compatibility
  • SVG icons are preferred for modern browsers (crisp at any resolution)
  • The logo height of 80px is recommended but can be adjusted for specific layouts
  • For a light-theme variant, please request a separate icon set
